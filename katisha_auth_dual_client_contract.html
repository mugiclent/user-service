Dual-client strategy — one route, two response modes

Web (no X-Client-Type header)

Tokens delivered as HttpOnly cookies. Body contains user object only. JS never touches tokens.

Mobile (X-Client-Type: mobile)

Tokens delivered in JSON response body. No cookies set. Client stores in secure enclave (Keychain / Keystore).

All endpoints share the same URL, same validation, same DB logic. The header is inspected only at the response-serialization step.
